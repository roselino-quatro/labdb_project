\section{Funcionalidades}
A plataforma foi desenvolvida considerando que as funcionalidades são herdadas entre os diferentes tipos de usuários herdam os acessos de seus tipos pai,
como por exemplo, o Funcionário do Cefer possui acesso a todas as rotas que um estudante da USP teria.

Dessa forma não vamos repetir as funcionalidades herdadas e listar individualmente apenas as que são novas para aquela classe


\subsection{Externo}
	\begin{itemize}
		\item ser convidado para o CEFER, incluindo aceitar e vizualizar atividades
	\end{itemize}

\subsection{Interno}
	\begin{itemize}
		\item Convidar usuários externos para atividades, listar convites passados e os status deles
		\item Visualizar as suas próprias reservas
		\item Ver e se inscrever em atividades
	\end{itemize}

\subsubsection{Funcionário do Cefer}
	\begin{itemize}
		\item Gerenciar atividades oferecidas
    \item Gerenciar a utilização dos equipamentos
    \item Gerenciar a utilização das instalações
    \item Gerenciar a particiação de externos em atividades
	\end{itemize}

\subsubsubsection{Educador do Cefer}
	\begin{itemize}
		\item Criar e gerenciar atividades
	\end{itemize}

\subsection{Administrador}
	\begin{itemize}
		\item Criar, deletar e atualizar usuários internos
		\item Criar, deletar e atualizar usuários externos
		\item Criar, deletar e atualizar funcionários
		\item Criar, deletar e atualizar educadores
		\item Criar, deletar e atualizar grupos de extensão
		\item Criar, deletar e atualizar equipamentos
		\item Criar, deletar e atualizar instalações
		\item Criar, deletar e atualizar eventos
	\end{itemize}
